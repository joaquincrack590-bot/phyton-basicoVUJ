# 1. ¿Qué es una Base de Datos Relacional?
# Una Base de Datos Relacional es un sistema que almacena información organizada en tablas que se relacionan entre sí mediante claves. Se basa en el modelo relacional propuesto por Edgar F. Codd, donde los datos se estructuran en filas y columnas.
# 2. ¿Qué es una Tabla en el contexto de Bases de Datos Relacionales?
# Una tabla es una estructura que organiza los datos en columnas (campos) y renglones (registros). Cada tabla representa una entidad, por ejemplo: Clientes, Productos o Empleados.
# 3. ¿Qué es una relación en el contexto de Bases de Datos Relacionales?
# Una relación es el vínculo que existe entre dos tablas mediante una clave primaria y una clave foránea. Permite conectar información de diferentes tablas sin duplicar datos.
# 4. ¿Qué representa una columna en el contexto de Bases de Datos Relacionales?
# Una columna representa un atributo o característica de la entidad.
# Ejemplo: en una tabla “Alumnos”, las columnas pueden ser: nombre, edad, matrícula.
# 5. ¿Qué representa un renglón en el contexto de Bases de Datos Relacionales?
# Un renglón (o fila) representa un registro completo, es decir, un conjunto de datos relacionados que describen una instancia específica.
# Ejemplo: un alumno específico con su nombre, edad y matrícula.
# 6. ¿Qué es un motor de base de datos relacional?
# Un motor de base de datos relacional es el software que permite crear, administrar y manipular bases de datos relacionales. Se encarga del almacenamiento, seguridad, consultas y control de acceso.

# 8. ¿Cuál es el motor de base de datos ligero que no necesita un servidor?
# El motor ligero que no necesita servidor es SQLite.
# Se usa mucho en aplicaciones móviles y programas pequeños porque funciona como un archivo local.
# 9. ¿Qué significa SQL en el contexto de bases de datos relacionales?
# SQL significa Structured Query Language (Lenguaje de Consulta Estructurado). Es el lenguaje estándar que se utiliza para crear, consultar y modificar datos en bases de datos relacionales.
